INSTRUKCJA FIRESTORE — ADS Viewer Pro V17

CEL
Synchronizacja danych programu między telefonem i komputerem.
Ten sam projekt Firebase + ten sam kod synchronizacji = te same zapisane samoloty, obserwowane samoloty i ustawienia programu.

WAŻNE
Dane firebaseConfig w aplikacji webowej nie są hasłem. To normalne, że są widoczne w kodzie strony.
Nie wpisuj tam żadnych prywatnych haseł.
Kod synchronizacji potraktuj jak prywatny identyfikator listy.
Od V17 program synchronizuje też ustawienia źródła danych/API, motyw, filtry, tryb wydajności, obserwowane samoloty oraz ustawienia alertów.

KROK 1 — UTWÓRZ PROJEKT FIREBASE
1. Wejdź na: https://console.firebase.google.com/
2. Kliknij: Add project / Dodaj projekt.
3. Nadaj nazwę, np. ADS Viewer Pro.
4. Google Analytics możesz wyłączyć, jeśli nie jest potrzebne.
5. Utwórz projekt.

KROK 2 — DODAJ APLIKACJĘ WEB
1. W projekcie kliknij ikonę Web: </>
2. Nazwa aplikacji, np. ADS Viewer Web.
3. Nie musisz konfigurować Firebase Hosting.
4. Po rejestracji Firebase pokaże obiekt firebaseConfig.
5. Skopiuj cały fragment:

const firebaseConfig = {
  apiKey: "...",
  authDomain: "...",
  projectId: "...",
  storageBucket: "...",
  messagingSenderId: "...",
  appId: "..."
};

KROK 3 — WŁĄCZ FIRESTORE
1. W menu Firebase wybierz Firestore Database.
2. Kliknij Create database / Utwórz bazę danych.
3. Wybierz tryb Native.
4. Wybierz lokalizację bazy.
5. Na start możesz wybrać Production mode, a potem wkleić reguły z kroku 4.

KROK 4 — WKLEJ REGUŁY FIRESTORE
W Firestore wejdź w zakładkę Rules / Reguły i wklej:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /adsViewerSync/{syncKey}/flights/{flightId} {
      allow read: if syncKey.matches('^[A-Za-z0-9_-]{8,80}$');
      allow create, update: if syncKey.matches('^[A-Za-z0-9_-]{8,80}$')
        && request.resource.data.syncKey == syncKey
        && request.resource.data.keys().hasOnly([
          'id', 'syncKey', 'deleted', 'deletedAt', 'icao', 'hex', 'name', 'callsign',
          'registration', 'type', 'aircraftType', 'kind', 'date', 'zoom', 'lat', 'lon',
          'altitude', 'speed', 'heading', 'routeShort', 'routeVerbose',
          'createdAt', 'updatedAt', '_clientId', '_updatedAtMs'
        ]);
      allow delete: if false;
    }
  }
}

Kliknij Publish / Opublikuj.

KROK 5 — SKONFIGURUJ APLIKACJĘ ADS VIEWER
1. Uruchom ADS Viewer Pro V17.
2. Przy pierwszym uruchomieniu pokaże się okno Firestore.
3. Wklej cały obiekt firebaseConfig.
4. Kliknij „Odczytaj konfigurację z wklejonego tekstu” albo poczekaj, aż pola same się uzupełnią.
5. Wpisz kod synchronizacji, np. ads-tomek-2026-xxx.
6. Kliknij „Zapisz i synchronizuj”.

KROK 6 — SKONFIGURUJ DRUGIE URZĄDZENIE
1. Na telefonie uruchom tę samą aplikację.
2. Wklej ten sam firebaseConfig.
3. Wpisz dokładnie ten sam kod synchronizacji.
4. Kliknij „Zapisz i synchronizuj”.

JAK TO DZIAŁA
- Aplikacja trzyma lokalną kopię danych w przeglądarce.
- Po włączeniu Firestore pobiera dane z chmury.
- Scala zapisane samoloty lokalne i chmurowe.
- Synchronizuje też obserwowane samoloty oraz ustawienia programu.
- Techniczny stan ustawień jest zapisywany jako dokument `__app_state` w tej samej kolekcji Firestore, dlatego podane reguły nadal pasują do programu.
- Potem zmiany są synchronizowane w tle.

GDZIE ZMIENIĆ KONFIGURACJĘ
Ustawienia programu → Synchronizacja Firestore → Konfiguruj.

GDZIE WYMUSIĆ SYNCHRONIZACJĘ
Ustawienia programu → Synchronizacja Firestore → Synchronizuj teraz.

GDZIE WYŁĄCZYĆ
Ustawienia programu → Synchronizacja Firestore → Wyłącz.
